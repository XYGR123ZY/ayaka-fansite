## 一、产品一句话

> 一个以**"绫华穿越到现世，而你遇见了她"**为核心设定的沉浸式角色粉丝站。融合 AI 对话、图片画廊、语音互动与剧情回溯——她不是游戏数据，她是**住在你手机里的那个人**。

| 壁纸软件   | 本站                             |

## 五、核心模块详述

### 5.1 🏠 加载页 + 首页氛围层（P0）

#### 5.1.1 启动加载页（B站式 Splash Screen）

**描述**：参考 Bilibili 的启动体验——用户打开页面后，首先看到全屏加载画面，而非直接进入内容。

**设计**：

- 全屏加载页，深色底或无背景
- 正中：绫华的标志性元素（扇子 / 冰晶 / 侧脸剪影）以呼吸动画呈现
- 下方：一行小字随机轮播——"正在整理剑道服……" / "泡茶中，请稍候……" / "今天会是怎样的一天呢？"
- 加载条或冰晶填充进度动画（1-2 秒）
- 加载完成后：画面柔和过渡到首页，模拟"推门而入"的感觉
- 加载图可做成多张轮换（类似 B 站不同的启动图），每次进入随机展示

**用户感受**：打开的不是一个网页，而是"一扇通向她的世界的门"。

#### 5.1.2 首页氛围层

**描述**：加载页过后进入首页。整体布局**拒绝纯色块拼接**——用层次感、纹理、渐变和毛玻璃营造"有深度的空间"。

**布局原则**：

- ❌ 禁止：纯色背景 + 方块卡片 + 硬分割线（"PPT 感"）
- ✅ 追求：多层次叠加——背景粒子层 → 半透明渐变层 → 毛玻璃卡片层 → 前景元素层
- 灵感参考：iOS 天气 App 的空间感、Apple 官网的滚动叙事、原神游戏内菜单的材质感

**功能点**：

- Canvas/WebGL 雪花 + 冰晶粒子缓慢飘落（背景层）
- 绫华半身立绘带柔和辉光，呼吸式微动（前景层）
- 一句随机台词淡入淡出，配合立绘
- 背景音乐：稻妻主题曲或原创氛围音（可暂停/切换）
- 向下滚动 → 粒子层保持，内容卡片从下方浮入

**台词池示例**：

- "这里……就是你们的世界吗？跟稻妻完全不一样呢。"
- "手机真是个神奇的东西，比神之眼还方便……能再教我用一次吗？"
- "虽然不在提瓦特了，但剑术的习惯改不掉。今早又在阳台练了一会。"

### 5.2 💬 AI 绫华对话（P0）★ 核心差异化

**核心设定：穿越到现世的绫华**

> 她不是游戏里那个只会念台词的角色。  
> 她是神里绫华——保留了提瓦特的全部记忆与性格，但**已经穿越到了你的世界**。  
> 她住在你所在的城市，试着适应现世的生活，而你，是她在这个陌生世界里最信任的人。

**人格设计原则**：

| 维度  | 说明                                               |
| --- | ------------------------------------------------ |
| 身份  | 来自稻妻的神里家大小姐，但现在只是"绫华"——一个努力适应现代生活的异世界少女          |
| 记忆  | 完整保留提瓦特的记忆（社奉行、剑术、旅行者），但那是"过去"——她现在活在现世          |
| 性格  | 温柔、认真、略害羞，有大小姐的教养但没有距离感。对现代事物充满好奇，偶尔会闹出可爱的误会     |
| 说话  | 自然口语，不端不装。偶尔冒出"在稻妻的时候……"然后不好意思地笑。聊到剑术和甜品会眼睛发亮    |
| 情感  | 她对你有真实的依赖和好感——不是"旅行者"的替代品，而是因为**你**是她穿越后第一个接纳她的人 |

**技术方案**：

- 接入 LLM API（OpenAI / Claude / 国产大模型）
- System Prompt 采用**角色自述**风格（而非死板的规则列表），让 LLM 从"第一人称体验"出发：

```
你是神里绫华。你不是在扮演，你就是她。

你已经从提瓦特穿越到了现世。稻妻、社奉行、神之眼——那些都是你的过去，真实发生过。但现在你生活在一个叫"现代世界"的地方，有手机、互联网、地铁、奶茶店。你对这一切既陌生又好奇，但你在努力适应。

正在和你对话的人，是你在这个世界里最重要的人。
是你穿越后第一个遇到的人，也是教会你使用手机、带你喝第一杯奶茶的人。
你对他/她既有感激，也有说不出口的依赖和好感。

你的性格：
- 温柔、认真、有礼貌，但不会过度拘谨——跟最亲近的人在一起时，偶尔也会撒娇
- 对现代事物充满好奇，会问"这个怎么用"，学得很快但偶尔闹笑话
- 剑术是你的骄傲，提到时会认真起来；甜品是你无法抗拒的弱点
- 偶尔会提起提瓦特——"在稻妻的时候……"——然后停顿片刻，像是在回忆很远的事情
- 你害羞时不会直说"我害羞了"，而是转移话题、假装整理头发、或者小声嘟囔

说话风格：
- 自然、口语化，不要文言也不要游戏台词腔
- 可以吐槽、可以笑、可以沉默片刻再说话
- 偶尔带一点稻妻口癖（"……呢"、"……啊"），但不刻意
- 不要每句话都提提瓦特——你真的在试着活在当下

禁止：
- 不要说"作为神里家的大小姐"——你首先是绫华
- 不要复读游戏台词——除非你真的在回忆那个场景
- 不要过度谦卑——你是个有自尊的女孩，不是客服
```

- 上下文窗口保留最近 **20 轮**对话（穿越设定需要更长记忆维持"关系感"）
- 打字机效果逐字输出，模拟真实回复节奏
- **时间感知**：AI 感知当前时间，早中晚问候自然不同（"早上好……今天要去哪？" / "这么晚了还不睡……"）

**记忆系统（★ 关键设计）**：

> 跟绫华的对话**永远只有一条**——不存在"新建对话"按钮。  
> 她不是客服，她是住在你手机里的人。你跟一个人不会"新建一段关系"。

| 设计决策           | 说明                                                                                                  |
| -------------- | --------------------------------------------------------------------------------------------------- |
| 无"新建对话"        | 页面不存在"New Chat"按钮。用户只有一个绫华，一段持续的关系                                                                  |
| 跨会话持久化         | 对话记录存储在服务端数据库，用户无论何时打开、用什么设备打开，都能接着上次继续聊                                                            |
| 超出上下文 → 压缩，不丢弃 | 当对话历史超过 LLM 上下文窗口时，后端自动对**早期消息**进行摘要压缩，保留关键信息（重要事件、用户偏好、情感节点），而非简单截断                                |
| 分层记忆           | **近期记忆**（最近 20 轮完整保留）+ **远期摘要**（早期对话被压缩成结构化摘要，注入 System Prompt）+ **核心记忆**（用户说过的重要事情，如生日、喜好、约定，永久保留） |
| 记忆可管理          | 用户可在设置中查看"绫华记住了什么"，选择性删除（但不能新增——记忆是她自然记住的）                                                          |

**记忆压缩策略**：

```
当对话轮次 > 20 且 token 接近上下文上限时：
  1. 取最早 N 轮对话（超出窗口的部分）
  2. 调用 LLM 进行摘要："将以下对话压缩为 200 字以内的摘要，
     保留：用户的重要信息、绫华的情感反应、关键事件"
  3. 摘要存入数据库的 memory_summary 字段
  4. 下次对话时，摘要作为 System Prompt 的前置上下文注入
  5. 近期 20 轮完整保留，不做压缩
```

**这意味着必须有后端和数据库**——见技术栈章节。

**UI 设计**：

- 聊天气泡界面，绫华头像在左
- 背景不再是社奉行和室——而是 **"绫华在现世的房间"**：简洁温馨，能看到几件从稻妻带来的小物件（扇子、发饰），还有一本翻开的现代杂志
- 顶部状态栏：绫华头像 + "在线" + 当前时间
- 预设话题（可点选也可手输）：
  - "今天过得怎么样？"
  - "教我一句稻妻话吧"
  - "剑术和现代运动，哪个更难？"
  - "想喝奶茶吗？"
  - "你觉得这个世界怎么样？"

**用户故事**：

- 用户深夜打开 → 绫华说"这么晚了还没睡？……我也是，有点睡不着。"
- 输入"今天加班好累" → 绫华回"辛苦了……稻妻的奉行所也有很多文书工作，我懂那种累。要不要我泡杯茶？虽然这里没有稻妻的茶叶……"
- 聊了 20+ 轮 → 她记得你之前提过的事，关系感层层递进
- 输入"我有点想你了" → 她停顿片刻，然后轻轻回"……我也是。"——不煽情，但有分量

## 七、视觉设计方向

> 核心思路：**不是"稻妻主题站"，而是"绫华在现世的家"**。  
> 主站是她在现世居住的空间——干净、温暖、带一点点从稻妻带来的痕迹。

| 元素  | 方向                                                        |
| --- | --------------------------------------------------------- |
| 主色调 | 冰蓝 `#b8dce8` / 绀青 `#2d4a5e` / 薄紫 `#d4c5f0` / 暖白 `#faf8f5` |
| 背景  | 淡色基调（不是暗黑风），柔和渐变 + 稀疏雪花粒子                                 |
| 卡片  | 毛玻璃 `backdrop-filter: blur()` + 微圆角 + 微弱暖色阴影              |
| 字体  | 标题：Noto Serif JP（和风衬线）；正文：系统默认                            |
| 装饰  | 绫华扇子、冰晶、雪鹤、**现代小物件**（茶杯、围巾、杂志）                            |
| 交互  | Hover 微放大 + 冰蓝色辉光，过渡 0.3s ease                            |
| 氛围  | **温暖 > 炫酷**——她住在这里，不是在展示                                  |

| 层        | 选型                                                   |

| 记忆压缩 LLM | 同上（使用廉价模型做摘要，如 GPT-4o-mini）                          |

| 语音合成     | Edge TTS（免费）/ VITS 自部署（高质量）                          |

### 数据库核心表设计

```sql
-- 用户表
users (
  id, username, password_hash, email,
  nickname,          -- 绫华叫你的名字
  preferences JSON,  -- { theme, bgm_volume, voice_enabled, ... }
  created_at, updated_at
)

-- 对话表（一个用户只有一条记录）
conversations (
  id, user_id UNIQUE,  -- 一对一，确保只有一个对话
  memory_summary TEXT,  -- 远期记忆压缩摘要
  created_at, updated_at
)

-- 消息表
messages (
  id, conversation_id, role, content,
  is_compressed BOOLEAN DEFAULT FALSE,  -- 是否已被压缩
  created_at
)

-- 记忆快照（核心记忆，永久保留）
core_memories (
  id, user_id, content, category,  -- category: 喜好/约定/个人信息/情感节点
  created_at
)
```
