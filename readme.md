# Context

我正在开发一个名为“神里绫华 · 沉浸式粉丝站”的项目。

- **核心设定**：“绫华穿越到现世，而你遇见了她”。她不是游戏数据，而是保留提瓦特记忆、努力适应现代生活、对你有真实依赖的立体少女。
- **产品定位**：融合 AI 对话、图片画廊、语音互动的多感官沉浸平台，拒绝传统壁纸软件的“被动浏览”和“用完即走”。
- **技术栈（基于准备清单）**：
  - 前端：Next.js (App Router) + Tailwind CSS + Framer Motion + Canvas/WebGL
  - 后端：Next.js API Routes
  - 数据库：SQLite (本地开发) / Supabase (生产环境)
  - AI 能力：OpenAI API (gpt-4o-mini) 用于对话，Edge TTS 用于语音合成。

# Constraints & Guidelines

1. **视觉与交互设计（极其重要）**：
   - ❌ 绝对禁止：纯色背景 + 方块卡片 + 硬分割线（拒绝“PPT感”）。
   - ✅ 必须追求：多层次空间叠加（背景粒子层 → 半透明渐变层 → 毛玻璃卡片层 → 前景元素层）。参考 iOS 天气 App 的空间感和原神游戏内菜单的材质感。
2. **代码规范**：
   - 提供可直接复制运行的完整核心代码，**严禁使用 `// ... 其余代码省略` 敷衍**。
   - 组件需具备良好的响应式设计（Mobile First）。
3. **AI 角色设定**：
   - 绫华的 System Prompt 必须采用 **“第一人称角色自述”** 风格，而非死板的规则列表。
   - 说话风格：自然口语，不端不装，温柔认真略害羞。偶尔提及“在稻妻的时候”，对现代事物（如手机、奶茶）充满好奇。
